1 . Summarize for us the goal of this project and how machine learning is useful in trying to accomplish it. As part of your answer, give some background on the dataset and how it can be used to answer the project question. Were there any outliers in the data when you got it, and how did you handle those?

The goal of the current project is to identify Enron employees who were involved in the Enron fraud based on their financial and email data. The dataset consists of features such as salary, bonus, stock options, number of emails, etc. The outcome, person of interest, is used as the label for training, testing, and evaluating the classifiers.

There are in total of 146 people entries and 20 features in the dataset. Among the 146 entries, 18 are classified as persons of interest. In other words, we are going to study the unique aspects of the 18 poi’s data in order to train the machine to identify fraud suspects on a high level.

Outliers in the dataset include a ‘Total’ entry that sums up all the employees’ features. It was identified through plotting the spread out of data points (see below). Another outlier is an entry called ‘the travel agency in the park’, which was found by reading the pdf file of insider pay information. Both of the two outliers were removed. Other seem-to-be outliers contain critical information, thus were kept for further usage.

Several features in the data have very high null percentage (over 60%), such as 'loan_advances', 'director_fees', 'restricted_stock_deferred', and 'deferral_payments'. However these features are not removed as they may play a key role in identifying poi's. I have also added three new features. 'ratio_to_poi' and 'ratio_from_poi' are the percentage of email interactions with poi across all the to and from emails.         if data_dict[name][denominator] == 'NaN' or data_dict[name][denominator] == 0 \

            data_dict[name][ratio_list] = 0
        if data_dict[name][denominator] != 'NaN' and data_dict[name][denominator] != 0 \

            ratio = data_dict[name][numerator] / float(data_dict[name][denominator])
            data_dict[name][ratio_list] = ratio

            data_dict[name][ratio_list] = 'NaN'

Feature scaling was performed only when it's required by the classifier (SVC and KNN).

The features used in my POI identifiers are selected using SelectKBest combined with plotting. The goal of plotting the scores is using visualization to identify the cut-off point in the number of features selected. The cutoff point is the point where marginal benefits of adding a new feature being the lowest. As we see that there's a sharp decrease in fit score right after 'salary', we know that the benefit of adding in the next feature is not as much as adding in any features in front of it. To maintain the most efficient selection of features, only the four features that have significantly higher benefit margin will be included.

The following features were selected using the method described above: 'exercised_stock_options', 'total_stock_value', 'bonus', 'salary'.

4 . What does it mean to tune the parameters of an algorithm, and what can happen if you don’t do this well? How did you tune the parameters of your particular algorithm? (Some algorithms do not have parameters that you need to tune -- if this is the case for the one you picked, identify and briefly explain how you would have done it for the model that was not your final choice or a different model that does utilize parameter tuning, e.g. a decision tree classifier).

The goal of tuning parameters is to find the optimal values to complete a task efficiently. Different parameters can produce a huge difference in the decision boundary the algorithm arrives at. Not tuning parameters well may suffocate the accuracy of the algorithm. In our project, for example, the classification of POI will be likely to be wrong or slow, not realizing the full potential of the classifiers.

In my project, I have used GridSearchCV to tune the parameters for DecisionTreeClassifier, SVC, and RandomForestClassifier. I would usually choose a set of possible ranges of parameter values and tested the models' accuracy, precision, recall, and f1 metrics. If the parameter chosen by the best estimator happens to be at the edge of the range, I would adjust the range to find a more suitable parameter.

I have also tuned the scoring attribute in GridSearchCV. Originally the random tree classifier failed to have a good recall score. After adjusting the scoring into 'recall', I was able to find the set of parameter that produced qualifying metrics.

5 . What is validation, and what’s a classic mistake you can make if you do it wrong? How did you validate your analysis?

Validation the process of ensuring a measurement correspond accurately to the real situation. The goal of cross-validation is to add another independent 'testing' stage in the training phase, in order to avoid overfitting of the data. In other words, it helps improve model prediction by averaging prediction errors. A classic mistake is using the same data for training and testing the classifier.

Cross-validation is usually done by splitting the data into training, (validation), and testing sets. In this project, because of the limited size of data (especially true POI's), I used stratified shuffle split to partition all data points before splitting them into different sets. In this way, a data point can have more than one chances to be used for either training or testing, which help produce a more solid classifier.

6 . Give at least 2 evaluation metrics and your average performance for each of them. Explain an interpretation of your metrics that says something human-understandable about your algorithm’s performance.

DecisionTreeClassifier Average Precision score = 0.338 Average Recall score = 0.322

Precision (positive predictive value) measures the exactness or quality, while recall (sensitivity or selectivity) is a measurement of completeness or quantity.

Precision describes how well the model generalizes (the ability to select target from non-target). It's calculated by dividing true_positives by all the positive results. In this project, precision is the proportion of the correctly identified POI's to all the enron employees in the test set. The higher the precision score is, the more confident we are about the classifier.

Meanwhile recall is the ratio of all correctly classified points to all correct points. It is calculated by dividing true_positives by all relevant events. In our case, recall is the number of correctly identified POI's out of all the POI's that exist in the dataset. So the higher the recall score is, the higher our ability is to recover the POI signal.

Because of the trade-off effect between recall and precision, we need to use f1 scores, the average between the two, when selecting algorithms.
